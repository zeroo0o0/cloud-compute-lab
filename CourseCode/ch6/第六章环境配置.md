## 环境配置

### docker与wsl

*也可以使用虚拟机完成所有的操作，可以自行探索，这里提供完全纯净的环境如何进行最小化的配置*

在win上安装docker desktop（下载时注意要勾选**Use WSL 2 instead of Hyper-V**）

安装完成后，打开 Docker Desktop 的设置 (Settings) -> Resources -> WSL Integration，确保你刚才安装的 Ubuntu 处于开启状态。

wsl的安装相当于模拟与linux和mac相同的操作：
```shell
wsl --install
```

安装对应的镜像：

```shell
wsl --install -d Ubuntu
```

如果不希望C盘占用过多，可以迁移到D盘，如（非必要不建议折腾，还需要修改docker desktop相关的配置，重新指向Settings->Resources->WSL Integration相关的配置）：
```shell
wsl --shutdown # 停机
wsl -l -v # 查看镜像名称
wsl --export Ubuntu D:\WSL_Backup\ubuntu.tar # 将当前系统保存备份
wsl --unregister Ubuntu # 注销现有系统，Ubuntu为系统名称
wsl --import Ubuntu D:\WSL\Ubuntu D:\WSL_Backup\ubuntu.tar --version 2 # 从备份导入
```

打开docker与wsl的管道：

docker desktop下**Settings->Resources->WSL Integratoin**下**Enable integration with additional distros:**下的Ubuntu按钮打开

在wsl中运行`docker version`来确定是否接通（wsl中不需要额外安装docker）

### 安装minikube与kubectl

#### wsl使用系统代理（win11）

**非常推荐使用代理，因为国内镜像极度容易出现问题**

在win下运行

```shell
notepad "$env:USERPROFILE\.wslconfig"
```

写入

```
echo "[user]" >> /etc/wsl.conf
echo "default=ashionial" >> /etc/wsl.conf
```

重启wsl

```shell
wsl --shutdown
wsl -d Ubuntu
```

此时wsl可以走系统代理

验证代理是否通入wsl：

```shell
curl -I https://www.google.com
```

如果不使用系统代理也可以完成实验，但是要寻找国内镜像，这些国内镜像会经常变动更新，并不稳定

#### 安装minikube

一个在单机可以模拟多机集群的工具，在wsl中安装如下：

```shell
 curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
 
 sudo install minikube-linux-amd64 /usr/local/bin/minikube
```

启动minikube

```shell
# 1. 启动 Minikube 集群（代理建议开启TUN模式，允许局域网连接，如果不行还可以设置防火墙）
minikube start --driver=docker --container-runtime=containerd

# 2. 开启监控插件 (HPA弹性实验所需)
minikube addons enable metrics-server
```

**如果始终会卡住，需要手动下载以下6个文件**

1. **kubeadm**:
   - [kubeadm 二进制文件](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubeadm)
   - [kubeadm.sha256 校验码](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubeadm.sha256)
2. **kubelet**:
   - [kubelet 二进制文件](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubelet)
   - [kubelet.sha256 校验码](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubelet.sha256)
3. **kubectl**:
   - [kubectl 二进制文件](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubectl)
   - [kubectl.sha256 校验码](https://dl.k8s.io/release/v1.35.1/bin/linux/amd64/kubectl.sha256)

在wsl中运行：

```shell
mkdir -p ~/.minikube/cache/linux/amd64/v1.35.1
cd ~/.minikube/cache/linux/amd64/v1.35.1
explorer.exe .
```

将上面6个文件拖入该目录中重新运行`minikube start --driver=docker --container-runtime=containerd`

如果还有问题，可以尝试下面的命令：
```shell
minikube delete # 清理缓存
sudo ip link set dev eth0 mtu 1350 # 调小WSL的网络包体积
```

安装成功后未来再次启动：
```shell
minikube start # 启动
minikube stop # 休眠
```

#### 安装kubectl

下载

```shell
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
```

安装到系统路径并赋予执行权限

```shell
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
```

验证是否安装成功

```shell
kubectl version --client
```

### 简单的验证

Kubernetes 本质上就是一个用 Go 写的超大型分布式状态机。我们接下来的操作，就是向它提交“期望状态”（比如：我需要几个容器跑起来），然后它的控制器会自动去调度和维持这个状态。

请在 Ubuntu / WSL 终端中依次执行以下命令：

#### 第一步：检阅你的“云端基地”

先看看集群节点是否健康：

Bash

```
kubectl get nodes
```

*（你会看到名为 `minikube` 的节点状态为 `Ready`，这就是你的单节点集群。）*

#### 第二步：一键部署你的第一个应用

我们用最经典的 Nginx 作为测试。告诉 K8s：“帮我跑一个 Nginx 容器！”

Bash

```
kubectl create deployment my-first-app --image=nginx
```

随时查看容器（Pod）是否已经跑起来了：

Bash

```
kubectl get pods -o wide
```

*（一开始状态可能是 `ContainerCreating`，稍等几秒就会变成 `Running`。）*

#### 第三步：体验秒级“弹性扩容”

假设现在突然涌入大量流量，一个 Nginx 扛不住了。在传统物理机时代你要买机器、装系统，但在 K8s 里，只需要一句话：

Bash

```
kubectl scale deployment my-first-app --replicas=3
```

敲完立刻运行下面这条命令观察：

Bash

```
kubectl get pods
```

*（你会发现，K8s 瞬间又拉起了两个全新的 Pod，总共 3 个副本在同时运行。这在后端高并发架构中是应对潮汐流量的神技。）*

#### 第四步：把服务暴露给外界访问

虽然容器跑起来了，但它们目前只在 K8s 内部的虚拟网络中互通。我们需要建立一个 Service，像负载均衡器一样把流量引进去：

Bash

```
kubectl expose deployment my-first-app --type=NodePort --port=80
```

**如何访问它？**

因为 WSL 网络的特殊性，K8s 提供了一个极其方便的后端调试命令——**端口转发 (Port Forward)**。它能直接把 K8s 内部的服务映射到你 Windows 的 `127.0.0.1`：

Bash

```
kubectl port-forward svc/my-first-app 8080:80
```

敲完这句命令后，终端会挂起显示 `Forwarding from 127.0.0.1:8080...`。

现在，打开你 Windows 电脑的浏览器，访问 `http://127.0.0.1:8080`，如果你看到了 `Welcome to nginx!` 的大字，这标志着你的流量成功穿透了 Windows -> WSL -> K8s Service -> 最终到达了 Pod 内部！

当你测试完，可以在终端按 `Ctrl + C` 停止端口转发。
