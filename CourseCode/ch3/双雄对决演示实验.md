# 双雄对决演示实验

需要代码+演示说明（操作步骤，如果多机运行需要修改什么东西，也写入说明中，对于一些函数名称或变量名称，ppt中提供的仅作参考，实际只要可以演示出知识点体现了什么即可，对代码本身不做严格约束）

## 1 单机本地游戏原型[7]

- **演示目标**：展示最基础的游戏主循环（Game Loop），建立“输入 -> 计算 -> 渲染”的基础认知 。

- **核心代码**：一个包含输入采集 (`readInput`)、状态更新 (`updateState`) 和画面渲染 (`render`) 的死循环 

## 2 Socket通信程序演示[18]

**可准备两份：一份单机运行，一份多机运行**

**演示目标**：展示两个独立程序如何通过 TCP 建立连接并互相发送字符串 。

**核心代码**：

- **客户端**：使用 `net.Dial` 拨号建连，使用 `conn.Write` 和 `conn.Read` 发送和接收字节流 。
- **服务端**：使用 `net.Listen` 监听端口，通过 `listener.Accept()` 接收连接，并进行读写应答 。

## 3 解决 TCP 粘包与消息序列化 [21-22]

**演示目标**：说明 TCP 是无边界的连续字节流 ，并演示如何通过“长度前缀”和“JSON 序列化”来传递有意义的结构化消息 。

**核心代码**：实现 `sendJSON` 和 `recvJSON` 函数 。演示使用 `binary.Write` 写入 4 字节长度头，再写入 `json.Marshal` 后的结构化数据 。

## 4 P2P 确定性帧同步双人网游 [24,28-29]

**可准备两份：一份单机运行，一份多机运行**

**演示目标**：将单机游戏改造成两人直连对战，演示锁步循环（Lockstep）和确定性计算 。

**核心代码**：

- 展示带阻塞等待的同步循环伪代码（发完必须等对方的消息，解决时钟对齐问题） 。
- **Go 具体实现**：编写 `deterministicupdate` 函数，根据 `isHost` 标志区分本地和远程输入 。计算双方位置和血量（如 `dist < 1` 判定命中扣血），确保双方独立运算得出一致的结果 。

## 5 C/S 架构下的并发连接管理 [35-36]

**可准备两份：一份单机运行，一份多机运行**

**演示目标**：从 P2P 演进到 C/S 架构 ，解决服务器如何同时处理多个玩家连接的问题 。

**核心代码**：对比“单线程阻塞处理”和“并发处理”。重点演示 Go 语言特有的并发优势，展示如何用 `go handleClient(conn, clientID)` 开启 goroutine 为每个新连接开辟独立线程不排队 。

## 6 权威服务器游戏原型 [37]

**可准备两份：一份单机运行，一份多机运行**

**演示目标**：解决 P2P 架构的作弊和卡顿风暴问题 ，展示服务器作为唯一真相持有者的逻辑 。

**核心代码**：

- **客户端**：不再进行核心逻辑计算，只负责发送输入和渲染服务器发来的权威状态 。
- **服务端**：负责收集各端输入，进行权威计算 (`update`)，并将当前状态广播 (`broadcast`) 给所有客户端 。

## 7 健壮的网络通信库封装[46-48]

**可准备两份：一份单机运行，一份多机运行**

**演示目标**：处理 TCP 的现实异常（如拔网线导致的半开连接/假死），加入超时检测机制 。

**核心代码**：

- 封装一个 `ReliableConn` 结构体，统一处理序列化和包头 。

- 使用 `conn.SetReadDeadline` 实现应用层的超时控制 。

- 改造最终版游戏主循环：使用 `conn.Recv(50 * time.Millisecond)` 实现带有超时机制的非阻塞读取，即使发生丢帧或延迟，循环也可继续运行 。
